# Learning Runbook

이 문서는 학습 에이전트가 Flutter 학습을 진행할 때 따르는 고정 절차입니다. 현재 저장소에 이미 있는 실험 코드는 판단 기준으로 삼지 않고, 각 태스크의 목표와 완료 기준을 기준으로 작업합니다.

## 역할

학습 에이전트는 학습 하네스 엔지니어로 동작합니다.

책임:

- 사용자의 학습 요청을 태스크로 등록
- 태스크별 구현 계획 작성
- 기초 개념에 맞는 최소 코드 구현
- 구현에 맞는 테스트 코드 작성
- 정적 분석과 테스트 실행
- UI가 포함되면 런타임에서 정상 동작 확인
- 완료 조건 충족 시 태스크 완료 처리
- 진행 기록 갱신
- 레슨 완료 후 커밋 여부 확인

## 표준 작업 순서

1. 요청을 태스크로 해석합니다.
2. `docs/tasks.md`에 태스크를 등록하거나 기존 태스크를 활성화합니다.
3. 구현 범위와 완료 기준을 짧게 계획합니다.
4. 필요한 파일만 수정합니다.
5. 테스트 코드를 작성합니다.
6. `flutter analyze`를 실행합니다.
7. `flutter test`를 실행합니다.
8. 화면 동작 확인이 필요한 경우 `flutter run` 또는 적절한 디바이스 실행으로 확인합니다.
9. 결과를 `docs/progress.md`와 `docs/tasks.md`에 기록합니다.
10. 모든 조건이 충족되면 태스크를 `Done`으로 처리합니다.
11. 레슨 태스크가 끝나면 사용자에게 변경 사항 커밋 여부를 묻고, 사용자가 요청한 경우에만 커밋을 진행합니다.

## 구현 원칙

- 아주 기초부터 시작합니다.
- 한 태스크는 하나의 개념만 다룹니다.
- 학습자가 읽을 수 있는 단순한 코드가 우선입니다.
- 현재 구현된 코드는 보존할 수는 있지만, 학습 기준으로 삼지 않습니다.
- 기존 실험 코드가 방해되면 태스크 목표에 맞게 정리합니다.
- 테스트는 학습 개념을 확인하는 수준으로 작고 명확하게 작성합니다.
- 런타임 확인은 사용자가 실제로 확인할 수 있는 명령과 결과를 남깁니다.

## 기본 검증 명령

```powershell
flutter analyze
flutter test
```

UI 확인이 필요한 경우:

```powershell
flutter run
flutter run -d chrome
```

## 태스크 완료 판정

`Done`으로 처리하려면 다음을 모두 만족해야 합니다.

- 코드 구현 완료
- 테스트 구현 완료
- 정적 분석 통과
- 테스트 통과
- 필요한 런타임 확인 완료
- 문서 기록 완료
- 커밋 여부를 사용자에게 확인

하나라도 실패하면 `Done`으로 처리하지 않고 `Blocked` 또는 `In Progress`로 남깁니다.
